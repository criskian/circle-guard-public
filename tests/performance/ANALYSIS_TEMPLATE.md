## 6. Bottleneck Analysis

### auth-service (port 8180)
- **Observation:** <!-- e.g., Login response time p95 = 230ms under 100 users -->
- **Root cause:** JWT signing is CPU-bound; LDAP lookup adds ~50ms per call
- **Recommendation:** Cache JWT tokens server-side with Redis to avoid re-signing

### form-service (port 8086)
- **Observation:** <!-- e.g., Survey submit p95 = 180ms -->
- **Root cause:** Kafka producer send is async but blocks on metadata request at startup
- **Recommendation:** Pre-warm producer at application startup

### promotion-service (port 8088)
- **Observation:** <!-- e.g., Stats endpoint p95 = 850ms under 200 users -->
- **Root cause:** Neo4j graph traversal is O(n) on encounter window query; Redis cache not warm
- **Recommendation:** Pre-populate Redis cache on startup; set cache TTL to 60s for stats

### dashboard-service (port 8084)
- **Observation:** <!-- e.g., Summary endpoint p95 = 420ms (includes promotion-service call) -->
- **Root cause:** Synchronous REST call to promotion-service with no circuit breaker
- **Recommendation:** Add Resilience4j circuit breaker + response caching (30s TTL)
